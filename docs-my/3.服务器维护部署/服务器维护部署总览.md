# 服务器维护部署总览

这组笔记用于整理常见的 Linux 服务器维护与部署实践，偏向 Ubuntu + Docker + Nginx 的常用场景。

## 目录

- [[Ubuntu 服务器使用]]
- [[Linux 包管理工具使用与区别]]
- [[Docker 与 Dockerfile 使用]]
- [[Nginx 默认配置与使用]]

## 适用场景

- 新服务器初始化
- 应用部署与重启
- Docker 镜像构建与容器运行
- Nginx 反向代理与静态资源托管
- 日常磁盘、内存、服务状态巡检

## 建议维护思路

1. 先把系统级配置固定下来，包括用户、SSH、时区、环境变量、开机自启。
2. 再把应用运行方式标准化，例如 `systemd`、`docker compose`、反向代理配置。
3. 最后补上自动化脚本和巡检命令，减少手动操作。

## 常用检查入口

```bash
uname -a
lsb_release -a
uptime
df -h
free -h
docker ps
systemctl --failed
nginx -t
```
