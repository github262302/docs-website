# 虚拟环境安装 `dashscope` 权限问题总结

本次问题出现在项目虚拟环境 `.venv` 中执行 `python -m pip install dashscope` 时，终端报错：`OSError: [Errno 13] 权限不够`，具体路径为 `.venv/lib/python3.12/site-packages/websocket`。这类错误并不是 `dashscope` 包本身有问题，而是当前虚拟环境目录的属主和权限异常，导致普通用户无法向 `site-packages` 写入文件。

排查后确认，`.venv` 及其中的安装目录属于 `root:root`，而当前实际使用的用户是 `loveroot`。这通常说明之前在虚拟环境相关操作中使用过 `sudo`，例如用 `sudo` 创建虚拟环境，或者执行过 `sudo python -m pip install ...`。一旦这样做，虚拟环境里的部分目录和文件就会被 root 接管，后续普通用户即使已经 `source .venv/bin/activate`，也仍然没有写权限，所以安装任何包都可能失败。

这个问题的根因不是“没有进入虚拟环境”，而是“进入了一个权限已经损坏的虚拟环境”。解决方法有两种。第一种是修复现有环境权限：执行 `sudo chown -R $USER:$USER .venv`，然后重新激活环境并安装依赖。第二种是更稳妥的做法，直接删除 `.venv` 后重新创建：`python3 -m venv .venv`，再执行 `python -m pip install dashscope`。

后续应避免在虚拟环境中使用 `sudo pip` 或 `sudo python -m pip`。虚拟环境的目的就是让当前用户在项目目录内独立安装依赖，不需要管理员权限。只要保证 `.venv` 由当前用户创建和维护，这类权限错误通常不会再次出现。
