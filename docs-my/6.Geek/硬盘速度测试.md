# 一、Windows（DiskSpd）

👉 专业测试工具，比你乱拷贝靠谱多了

## 1️⃣ 大文件顺序读写（模拟磁盘吞吐）

### 顺序写（大文件）

```bash
diskspd -c10G -d30 -Sh -w100 -L -o1 -t1 -b1M test.dat
```

### 顺序读

```bash
diskspd -d30 -Sh -w0 -L -o1 -t1 -b1M test.dat
```

👉 参数重点：

* `-b1M`：大块 = 测大文件
* `-w100`：100%写
* `-o1 -t1`：单线程（纯盘性能）

---

## 2️⃣ 小文件随机读写（核心！）

### 4K 随机读写（最真实）

```bash
diskspd -c2G -d30 -Sh -w50 -L -o32 -t4 -b4K -r test.dat
```

👉 关键：

* `-b4K`：小文件/IO
* `-r`：随机
* `-o32 -t4`：高并发（模拟程序压力）
* `-w50`：读写混合

---

# 二、Linux（dd）

👉 dd 很原始，但够用

---

## 1️⃣ 大文件顺序写

```bash
dd if=/dev/zero of=test.dat bs=1M count=10240 oflag=direct
```

## 2️⃣ 顺序读

```bash
dd if=test.dat of=/dev/null bs=1M iflag=direct
```

👉 关键：

* `bs=1M`：大块
* `direct`：绕过缓存（真实磁盘）

---

## 3️⃣ 小文件随机（dd 做不了，要 hack）

dd 本身❌不适合小文件随机IO
你要这样搞👇

### 方法1：模拟小文件写

```bash
for i in {1..10000}; do
  dd if=/dev/urandom of=./small/file_$i bs=4K count=1
done
```

### 方法2：随机读

```bash
find ./small -type f | xargs -n1 -P4 cat > /dev/null
```

👉 这才接近真实小文件场景（inode + IO + 并发）

---

# 三、结论（直接说重点）

* **大文件**

  * Windows 👉 DiskSpd（1M block）
  * Linux 👉 dd（1M block）

* **小文件**

  * Windows 👉 DiskSpd（4K + 随机）
  * Linux 👉 ❌dd不行 → 用脚本模拟
